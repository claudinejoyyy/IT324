Lesson 4 - HTTP Request and Response Messages

HTTP Request Message

Request Line

  • The first line of the header is called the request line, followed by optional request headers.
    • The request line has the following syntax: request-method-name request-URI HTTP-version
      • request-method-name: HTTP protocol defines a set of request methods, e.g., GET, POST, HEAD, and OPTIONS. The client can use one of these methods to send a request to the server.
      • request-URI: specifies the resource requested.
      • HTTP-version: the version of the HTTP being used

Request Headers

  • The request headers/request methods are in the form of name:value pairs.
  • Multiple values, separated by commas, can be specified using request-header-name: request-header-value1, request-header-value2, ...

HTTP Response Message

Status Line

  • The first line is called the status line, followed by optional response header(s).
    • The status line has the following syntax: HTTP-version status-code reason-phrase
      • HTTP-version: The HTTP version used in this session. Either HTTP/1.0 and HTTP/1.1.
      • status-code: a 3-digit number generated by the server to reflect the outcome of the request.
      • reason-phrase: gives a short explanation to the status code.

Response Headers

  • The response headers are in the form name:value pairs: response-header-name: response-header-value1, response-header-value2, ...
  • Multiple values, separated by commas, can be specified using request-header-name: request-header-value1, request-header-value2, ...

Response Message Body

  • The response message body contains the resource data requested.
