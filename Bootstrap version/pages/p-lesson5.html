    HTTP Request Methods

    HTTP Request Methods

    • HTTP defines a set of request methods to indicate the desired action to be performed for a given resource. Although they can also be nouns, these request methods are sometimes referred as HTTP verbs.

    GET

    • This request method is the most commonly used HTTP method
    • This is used to request from the server the retrieval of the source identified by the request URI
      • The retrieved resource is returned in the message body as an entity
    • This is an idempotent method, along with HEAD, PUT, and DELETE.

    HEAD

    • This request method is similar to the GET response mesage, however the message body is not included in the response
    • This is an idempotent method, along with GET, PUT, and DELETE.

    POST

    • This request method requests that the origin server accepts the entity enclosed in the request as a new subordinate of the resource identified by the Request-URI in the Request-Line.

    OPTIONS

    • This request method requests for information about the communication options available on the request/response chain identified by the Request-URI.
    • This method allows the client to determine the options and/or requirements associated with a resource, or the capabilities of a server, without implying a resource action or initiating resource retrieval

    TRACE

    • This request method requests the server to "echo" back to the client the received request
    • This method is typically used for testing/diagnostics of the request chain

    PUT

    • This request method requests the server to store the enclosed entity in the message under the specified request URI

    DELETE

    • This request method requests the server to delete the resource identified by the request URI

    CONNECT

    • This request method is reserved for use of tunneling proxy servers
