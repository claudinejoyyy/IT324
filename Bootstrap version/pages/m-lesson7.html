    Lesson 14

    Client-side Scripting with JavaScript

    JavaScript

    • HTML and CSS have little interactivity (passive). This is where JavaScript comes in.
    • Associates a program with the web page
    • A program/script is downloaded and executed in the browser

    Associating JavaScript

    1. Externally linked (can be placed anywhere)
      • <script type = "text/javascript" src = "script.js">
    2. Embedded within HTML
      • <script>
	console.log('embedded script...')
</script>
    3. In-line script
      • <button onclick = console.log (“inline script…”);>Click me</button>
    • Code defined outside of a function is global or top level.
    • Code defined inside of a function is function body

    Attributes

    • Defer attribute
      • Code does not execute until document has been rendered
      • Fetcing does not block the rendering
    • Async attribute
      • Fetch the script, render the page, then execute even the if rendering is not yet done.
      • Some browsers do not support defer and async, in this case, placing will be the key
      • For browsers with no scripting support or with scripting support disabled: <noscript> No Scripting Support. </noscript>
    • Some browsers disable JavaScript for security measures purposes.

    Window object

    • Represents a browser environment
    • To manipulate a document:
      • window.document or document
    • To return a reference by element:
      • window.document or document
    • To set an attribute:
      • h.setAttribute(‘title’)
    • To manipulate the styling:
      • h.style = “color: blue”
    • To retrieve all elements of a familiar style
      • [p, p, p, p]
    • querySelector()
      • Argument will be a CSS selector
      • If there is more than one match, it will choose the first
    • querySelectorAll()
      • Will return all matches except pseudo elements
      • document.querySelectorAll(‘body p’)
      document.childNodes The childNodes property returns a collection of a node's child nodes, as a NodeList object. The nodes in the collection are sorted as they appear in the source code and can be accessed by index numbers. The index starts at 0. Note: Whitespace inside elements is considered as text, and text is considered as nodes. Comments are also considered as nodes.
      document.children The children property returns a collection of an element's child elements, as an HTMLCollection object. The elements in the collection are sorted as they appear in the source code and can be accessed by index numbers. The index starts at 0. The difference between this property and childNodes, is that childNodes contain all nodes, including text nodes and comment nodes, while children only contain element nodes.
      document.haschildNode The hasChildNodes() method returns true if the specified node has any child nodes, otherwise false. Note: Whitespace inside a node is considered as text nodes, so if you leave any white space or line feeds inside an element, that element still has child nodes.
      document.body The body property sets or returns the document's body. On return, this property returns the body element of the current document. On set, this property overwrites all child elements inside the existing body element, and replaces it with the new, specified content.
      document.head.childNodes Returns the head element of the current document. If there are more than one head elements, the first one is returned.
      document.head.children The same as the document.body.children, but this time it is in the head element.
      document.body.firstChild The firstChild property returns the first child node of the specified node, as a Node object. The difference between this property and firstElementChild, is that firstChild returns the first child node as an element node, a text node or a comment node (depending on which one's first), while firstElementChild returns the first child node as an element node (ignores text and comment nodes).
      document.body.firstElementChild The firstElementChild property returns the first child element of the specified element.
      document.body.firstChild.nextSibling The nextSibling property returns the node immediately following the specified node, in the same tree level. The returned node is returned as a Node object.
      document.body.firstChild.previousSibling The previousSibling property returns the previous node of the specified node, in the same tree level. The returned node is returned as a Node object.
